# 题目颗粒化重构方案

## 背景

目前每套真题/模拟卷是一个巨大的 `content.md` 文件（2009~2025 共 17 个 + 8 套模拟卷 = ~1000 道题挤在 25 个文件中）。问题：
- 增删改一道题需要编辑整个大文件
- 无法按科目/年份/知识点灵活筛选
- 题型混排（选择题+解答题）结构化差

## 目标

每道题独立为一个文件，通过 frontmatter 标签（年份/科目/知识点/题型）实现**多维度筛选**并支持**分科/分年/分知识点练习**。

## 一、内容架构

### 1.1 目录结构

```
content/
  question/                              # 所有题目扁平存放
    _index.md                            # 可筛选的题目列表页
    2009-ds-001.md                       # 命名: {year}-{subject}-{number}
    2009-ds-002.md
    2009-co-011.md
    2009-os-023.md
    2009-cn-033.md
    2009-ds-041.md                       # 大题也用同一个命名规则
    ...
    2025-cn-040.md
    simulate-001.md                      # 模拟题用 simulate- 前缀
    simulate-002.md
    ...
```

### 1.2 科目缩写

| 缩写 | 科目 |
|------|------|
| ds | 数据结构 |
| co | 组成原理 |
| os | 操作系统 |
| cn | 计算机网络 |

### 1.3 每个题目的 frontmatter

```yaml
---
title: "2009 数据结构 第1题"
date: 2026-07-01
type: question
years: ["2009"]
subjects: ["数据结构"]
knowledge_points: ["数据结构"]
question_type: "choice"             # choice | comprehensive
difficulty: 2
source: "408真题"                    # 或 "模拟题"
number: 1
related_knowledge:                  # 关联知识点路径（可选）
  - "/docs/data-structure/stack/"
---
```

## 三、配置文件改动

### 3.1 Hugo 分类法（hugo.yaml）

```yaml
taxonomies:
  author: authors
  tag: tags
  publication_type: publication_types
  year: years
  subject: subjects
  knowledge_point: knowledge_points
```

新增了 `year`、`subject`、`knowledge_point` 三个分类法。Hugo 自动生成：
- `/years/2009/` —— 列出所有 2009 年题目
- `/subjects/数据结构/` —— 列出所有数据结构题目
- `/knowledge_points/栈/` —— 列出栈相关知识题

### 3.2 导航（menus.yaml）

```yaml
main:
  - name: 首页      url: /
  - name: 知识点    url: /docs/
  - name: 刷题      url: /exam/     # 统一入口，覆盖真题/试题/模拟卷
  - name: 知识图谱  url: /graph/
  - name: 标签      url: /tags/
  - name: 资源      url: /resources/
```

移除独立的「真题」和「试题」导航项，统一到「刷题」。

## 七、迁移脚本

使用 Python 脚本批量处理 `content/exam/408quiz/{year}/content.md`：

1. 用 `##### N` 正则切分题目
2. 对每道题提取题号、正文、选项、`[tag_link]`、答案、解析
3. 根据题号范围判断所属科目
4. 生成 frontmatter（年份、科目、题型、难度、标签）
5. 写出到 `content/question/{year}-{subject}-{number}.md`

大题（41~47 题，无 `正确答案：X` 选项）同样独立成文件，`question_type: "comprehensive"`。

---

## 八、注意事项

- **大题答案折叠**：依赖 `[tag_link]` + `正确答案：` 正则匹配（或 fallback `[tag_link]` 标记）
- **知识点标签**：初始迁移时 `knowledge_points` 可能为空，需后续补充
- **模拟卷编号**：使用 `simulate-{set}-{num}.md` 避免与真题编号冲突
- **性能**：1000+ 题目文件 Hugo 构建无压力（当前已有 332 页面 + 3238 静态文件，构建正常）
