### 3.7a 选择题渲染规则（详细）

**内容文件格式要求**：

```markdown
题干文本，可以包含 $LaTeX$ 和 Markdown 链接。

[知识点标签](/path/to/tag)

A\. 选项一
B\. 选项二
C\. 选项三
D\. 选项四

[tag_link]

正确答案：A

分析文本可以多行，支持 Markdown。
```

**关键规则**：

1. **选项必须使用 `\.` 转义句点**（避免 Goldmark 误认为有序列表）
2. **4 个选项之间不要有空行**（保持在同一 `<p>`，JS 多选项拆分正则 `/([A-D])\s*\.\s*(.*?)(?=\s*[A-D]\s*\.|$)/g` 统一解析）
3. **`[tag_link]` 独占一行**，前后各空一行：
   - 前空行：使 `[tag_link]` 与选项分离
   - 后空行：使 `[tag_link]` 与 `正确答案` 分离
4. **`正确答案：X` 独占一行**，后空一行：
   - 使分析文本成为独立的 `<p>`，被 `findAnswer` 移入 `.answer-panel`
5. **`正确答案：X` 与分析文本之间必须有空行**（否则 Goldmark 同段落内文本丢失）

**JS 解析流程**：
1. 扫描所有 `<p>`，`textContent.trim() === '[tag_link]'` 的隐藏（`display: none`）
2. 在每个 `.question-block` 中搜索 `正确答案[：:]\s*([A-D])`：
   - 拆分文本节点，创建 `.answer-panel`（初始 `display: none`）
   - 将 `<p>` 的后续兄弟节点移入面板
   - 替换 `<p>` 为面板
3. 扫描所有 `<p>` 直接子节点，匹配 `([A-D])\s*\.\s*(.*?)(?=\s*[A-D]\s*\.|$)`：
   - 同一 `<p>` 内可含多个选项行（无空行），正则拆分
   - 每个匹配项生成 `<div class="exam-option">`
4. 替换 `【解析】` 为 `<div class="analysis-title">解析：</div>`

### 3.7b 大题（综合题）渲染规则

**内容文件格式要求**：

```markdown
题干文本，可以包含多段描述和子问题。

(1) 子问题一
(2) 子问题二

[tag_link]

1）答案第一点...注意不同小问的回答务必回车隔开

2）答案第二点...

【另解】其他解法...
```

**关键规则**：

1. **`question_type` 必须设为 `"comprehensive"`**（区别于 `"choice"`）
2. **`[tag_link]` 独占一行**，前后各空一行
3. **无 `正确答案：X` 行**（JS 检测不到时触发综合题分支）
4. **`[tag_link]` 后所有内容**被移入 `.answer-panel`

**JS 解析流程**：
1. 同选择题，先隐藏 `[tag_link]` 段落
2. `findAnswer` 搜索 `正确答案` → 未找到（`return false`）
3. 触发综合题分支：
   - 在 `.q-body` 中找最后一个文本精确为 `[tag_link]` 的 `<p>`
   - 创建 `.answer-panel`（`data-answer=""`，标题"答案与解析："）
   - 将 `[tag_link]` 的后续兄弟节点全部移入面板
   - 面板追加到 `.q-body` 末尾
4. 点击「查看答案」切换 `.answer-panel` 显示（无选项高亮）

### 8.1 三标签架构

| 标签 | data-graph | 数据来源 | 说明 |

### 8.2 数据流

```
静态构建时:
  content/question/questions/*.md  →  Python 爬虫脚本  →  question-kp-mapping.json
  content/question/tags/*.md                                           ↓
  模拟题知识点: 基于题号 + 科目模糊匹配 ←───────┘     static/mapping/question-kp-mapping.json

  layouts/partials/knowledge-graph-data-exam.html → 轻量 nav 配置（年份列表）
  layouts/partials/knowledge-graph-data-simulate.html → 同上（模拟卷列表）

运行时:
  用户点击「真题」标签
  → JS fetch('/cc408/mapping/question-kp-mapping.json')
  → buildNavigation() 构建 yearsMap / subjectsMap / kpMap
  → renderGraph() 用 D3 绘制
```

### 8.4 下钻层级（真题/模拟题）

```
年份模式 (view='year'):
  Level 0:            年份列表        getYearLevel()
    ↓ 点击年份
  Level 1:        [选择题] [综合题]   getYearQtypeLevel()
    ↓ 点击某题型
  Level 2:     科目（按题型过滤）      getYearQtypeSubjectsLevel()
    ↓ 点击某科目
  Level 3:        知识点球            getKpLevel()
    ↓ 点击某知识点
  Level 4:        题目叶子节点         getQuestionLevel()
    ↓ 点击题目球
  动作: 打开题目详情链接 (新窗口) + 显示信息面板

科目模式 (view='subject'):
  Level 0:            4 科            getSubjectLevel()
    ↓ 点击某科
  Level 1:          年份列表           getSubjectYearsLevel()
    ↓ 点击年份 → 切换到年份模式继续下钻
```

### 8.5 题型过滤规则

```
选择题 (selection):  题目编号 ≤ 40
综合题 (comprehensive): 题目编号 > 40  (即 41-47)

每层按 state.focus.qtype 过滤，只显示匹配该题型的子节点
```

### 8.6 节点类型与渲染

| type | 半径 | 颜色 | 点击行为 |

## 8. 内容完整性校验规则

### 8.1 选择题必填字段

每个选择题 `.md` 文件必须包含以下内容，缺一不可：

| 字段 | 校验规则 | 检测方法 |
|:-----|:---------|:---------|
| 题干 | body 非空，长度 ≥ 20 字符 | `body.strip()` 非空 |
| 4 个选项 | `A\.` `B\.` `C\.` `D\.` 各出现一次 | `re.findall(r'^[A-D]\\.', text)` 计数 = 4 |
| 正确答案 | `正确答案：X` 行存在 | `re.search(r'正确答案[：:]\s*[ABCDabcd]', text)` |
| 解析 | `正确答案` 后有非空分析文本 | 去除答案行后文本 ≥ 5 字符 |

**选项格式检查**：
- 必须使用 `\.` 转义句点（禁用 `A.` 未转义）
- 4 个选项间无空行（保持同一段落）
- 选项文本不为空

### 8.2 综合题必填字段

| 字段 | 校验规则 | 检测方法 |
|:-----|:---------|:---------|
| 题干 | body 非空，长度 ≥ 20 字符 | `body.strip()` 非空 |
| `[tag_link]`标记 | 存在 `[tag_link]` | `'[tag_link]' in text` |
| 解析 | `[tag_link]` 后非空 | `len(text[after_tag:].strip()) >= 10` |

### 8.3 批次修复流程

```
1. check_years.py        → 扫描 [tag_link]/正确答案 空行格式
2. validate_content.py   → 扫描 选项/题干/解析 完整性
3. restore_missing.py    → 从 crawled_data/ 恢复缺失内容
4. resolve_duplicates.py → 清理 ds/co 重复，统一 subject 名称
```
