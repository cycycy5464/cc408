# 真题修复流程

> 处理某一年真题的标准操作流程。
> 执行前先读 [修复标准.md](修复标准.md) 了解所有格式化约束。

---

## 阶段 1：准备

```bash
YEAR=2011   # ← 替换为实际年份
```

确认数据源存在：
- 爬虫数据：`tasks/408-crawler/crawled_data/${YEAR}.json`
- 题目目录：`content/question/` 下存在 `${YEAR}-*.md` 共 47 个文件

---

## 阶段 2：完整性检查

### 2.1 文件数量

408 真题每卷 47 题：40 选择 + 7 综合（41~47）。

```bash
ls content/question/${YEAR}-*.md | wc -l
# 应为 47
```

> 若不足 47，从爬虫数据提取缺失题目（`restore_missing.py`）。
> 若多于 47，检查是否有 科目重复文件（见 [修复标准#重复文件](修复标准.md#四重复文件))。

### 2.2 内容完整性

```bash
python scripts/validate_content.py ${YEAR}
```

检查项：
- 选择题：4 个选项、`正确答案`、非空解析
- 综合题：`[tag_link]`、非空解析

### 2.3 选项缺失恢复

若 `validate_content.py` 报告某题 options count < 4，说明选项被爬虫源数据覆盖丢失，需从爬虫 JSON 恢复：

```bash
python scripts/restore_missing.py ${YEAR}
```

恢复后运行 `validate_content.py` 再次确认 4 选项齐全。

### 2.4 泄露内容检查

扫描选择题解析末尾是否有下题内容泄露：

```bash
grep -n '组成原理 [0-9]\|操作系统 [0-9]\|计算机网络 [0-9]' content/question/${YEAR}-*.md
```

若泄露，手动删除从泄露标记到文件末尾的内容（不要用 git restore）。

## 阶段 3：格式修复

### 3.1 换行符统一

```bash
# content/question/ 下所有 md 文件
python -c "
import glob
for fp in glob.glob('content/question/*.md'):
    with open(fp, 'rb') as f:
        raw = f.read()
    new = raw.replace(b'\\r\\r\\n', b'\\n').replace(b'\\r\\n', b'\\n')
    if new != raw:
        with open(fp, 'wb') as f:
            f.write(new)
        print(fp)
"
```

爬虫 `4cf9d6c` 版本的源文件使用 `\r\r\n`（双回车换行），在 Linux 构建时会触发 Hugo panic。必须统一为 `\n`。

### 3.2 选择题选项空行

选项间不能有空行，否则 Goldmark 拆分为独立 `<p>`。（略，同前）

### 3.3 综合题解析分段

综合题解析的子问题（1）、2）、3）等）之间必须空行分开，不能挤在同一行。

**检查**：
```bash
# 查找子问题间无空行或同一行挤多个子问题的文件
python -c "
import re, glob
for fp in glob.glob('content/question/${YEAR}-*.md'):
    c = open(fp).read()
    if 'comprehensive' not in c: continue
    after = c[c.find('[tag_link]'):]
    lines = after.split(chr(10))
    sq = [i for i,l in enumerate(lines) if re.match(r'^\\s*[1-9][)）]', l)]
    for j in range(len(sq)-1):
        if not any(l.strip()=='' for l in lines[sq[j]+1:sq[j+1]]):
            print(f'{fp}: no blank between SQ{sq[j]}->SQ{sq[j+1]}')
    for i in sq:
        if re.search(r'）[^\\n]*）', lines[i]):
            print(f'{fp}: L{i}: multiple SQs on one line')
"
```

**修复**：手动在子问题之间插入空行，或将同一行的 `1）...2）...` 拆分为多行。

### 3.4 综合题 [tag_link]

每一道综合题（`question_type: "comprehensive"`）必须在知识标签后、解析前插入 `[tag_link]`：

```markdown
[知识标签](/study_methods/tags/...)
[知识标签2](/study_methods/tags/...)

[tag_link]

1）解析内容...
```

**关键**：
- `[tag_link]` 与解析之间必须空一行（否则 Goldmark 合并为同一 `<p>`）
- 每道题 **只能有 1 个** `[tag_link]`
- 若有旧版重复的 `[tag_link]`，只保留最前面的那一个

### 3.4 表格间距

表格前后必须有空行，否则 Goldmark 将后续文本吸入表格行：

```markdown
以下为计算结果。

| 列1 | 列2 |
| --- | --- |
| 数据 | 数据 |

由以上表格可知...
```

**检查**：
```bash
python -c "
import glob
for fp in glob.glob('content/question/${YEAR}-*.md'):
    lines = open(fp).read().split('\n')
    for i, l in enumerate(lines):
        s = l.strip()
        if s.startswith('|') and '|' in s[1:-1]:
            if i>0 and lines[i-1].strip() and not lines[i-1].strip().startswith('|'):
                print(f'{fp}:{i+1} text before table')
            if i+1<len(lines) and lines[i+1].strip() and not lines[i+1].strip().startswith('|'):
                print(f'{fp}:{i+1} text after table')
"
```

### 3.5 代码块语言标记

**规则**：
- 已有 ` ```c ` 标签的 → **不要动**（保持原样）
- 仅 ` ``` `（无语言标签）且内容为 C 代码的 → 改为 ` ```c `
- assembly、伪代码等非 C 内容 → 保持 ` ``` ` 不变

C 代码判据（满足任一即可）：
```
int / char / void / struct / typedef / return / if / else / for / while
// 注释 / { / }; / #include
```

### 3.6 爬虫残留

删除选择题文件末尾的 `### 解答题` 或 `## 解答题` 标题行。

```bash
grep -n '### 解答题\|## 解答题' content/question/${YEAR}-*.md
```

### 3.7 表格表头与分隔行空行

表头行与分隔行之间**不能**有空行（否则 Goldmark 不识别为表格）：

```markdown
# ❌ 错误
| 列1 | 列2 |

| --- | --- |

# ✅ 正确
| 列1 | 列2 |
| --- | --- |
```

**检查**：
```bash
python -c "
import glob
for fp in glob.glob('content/question/${YEAR}-*.md'):
    lines = open(fp).read().split('\\n')
    for i, l in enumerate(lines):
        if l.strip().startswith('|') and '---' in l:
            if i>0 and lines[i-1].strip().startswith('|') and not '---' in lines[i-1]:
                if not lines[i-2].strip():
                    print(fp + ':' + str(i+1) + ': blank line between header and separator')
"
```

### 3.8 零宽空格清理

爬虫数据中会混入零宽空格 `\u200b`，出现在矩阵、ASCII 图、代码段中，影响排版。

**检查**：
```bash
grep -rn '\xe2\x80\x8b' content/question/${YEAR}-*.md
```

**修复**：
```bash
python -c "
import glob
for fp in glob.glob('content/question/${YEAR}-*.md'):
    with open(fp, encoding='utf-8') as f:
        text = f.read()
    new = text.replace('\u200b', ' ')
    if new != text:
        with open(fp, 'w', encoding='utf-8') as f:
            f.write(new)
        print(fp)
"
```

### 3.9 综合题跨题污染

爬虫提取大题解析时，题目边界不精确导致跨题污染：
- 41 文件末尾含 42 内容
- 42 文件末尾含 43 内容
- 46 文件末尾含 47 内容

**检查**：
```bash
# 查找当前题号+1的内容混入
grep -n '第 42 题\| 42题' content/question/${YEAR}-041.md
grep -n '第 43 题\| 43题' content/question/${YEAR}-042.md
```

**修复**：截断至对应题号的污染标记前。

### 3.8 JS 注入污染

爬虫大题解析中混入整段 JS（`window.quizDB`、`IndexedDB`、`toggleSolutionDetail`）：

```bash
grep -l 'quizDB\|indexedDB' content/question/${YEAR}-*.md
```

**修复**：截断至 `if(typeof window.quizDB` 之前。

### 3.9 解析区尾部 UI 文本

解析区末尾出现 `查看答案与解析` / `收藏` / ` < 收藏` 等前端 UI 文本：

```bash
grep -n '查看答案与解析\|收藏' content/question/${YEAR}-*.md
```

**修复**：删除从 UI 文本到行/文件末尾之间的内容。

### 3.10 解析内容丢失

部分题目爬虫数据中无解析内容（`solutions[40]` 等为空）。

**修复**：
1. 检查爬虫 JSON 的 `solutions` 和 `explanations` 字段
2. 若均无有效内容，根据题干手动补写解析
3. 记录到已知问题列表

### 3.11 数据散落不成表

综合题中的表格数据（LSI 表、机器码表、路由表等）以纯文本逐行散布，未格式化为 MD 表格。

**修复**：
1. 识别数据行的表格结构（列数、表头）
2. 构造标准 MD 表格格式（表头行、分隔行、数据行）
3. 确保每行竖线数量统一
