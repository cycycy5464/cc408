# 真题修复 — 经验教训汇总

> 踩坑记录与排查指南。修复前先读 [修复标准.md](修复标准.md) 和 [修复流程.md](修复流程.md)。

---

## 1. 选项分隔符只认半角句点 `.`

JS 正则（`layouts/exam/408quiz-detail.html`）：

```javascript
var optParts = txt.match(/([A-D])\s*\.\s*(.*?)(?=\s*[A-D]\s*\.|$)/g);
```

- **只能匹配** `A.`、`B.` 格式（半角句点）
- **不能用** `、`（顿号）或 `．`（全角句点）
- 的原因：选项文本内可能包含 `、`（如 `PCI、CRT、USB、EISA`），`[A-D]\s*[.、．]` 会误匹配 `USB、EISA` 中的 `B、`

**教训**：2026-07-11 将 `[.、．]` 改为 `\.`。不要修改模板正则来兼容特殊格式，源文件用 `A\. ` 统一格式。

---

## 2. 批量脚本误删选项行（`c3c8178`）

**现象**：`content/question/` 下部分选择题缺失 A/B/C/D 选项。

**根因**：批量脚本直接全局正则替换，未做小范围试跑 + 人工验证。

**教训**：所有批量操作前必须先试跑 2~3 个文件并验证结果。

**恢复方法**：不要从 git 恢复（会覆盖其他修复），用 `restore_missing.py` 从爬虫 JSON 补充选项。

---

## 3. 双回车换行符导致 Linux 构建 panic

**现象**：GitHub Actions 构建时报 `panic: runtime error: slice bounds out of range`。

**根因**：爬虫源文件使用 `\r\r\n`（双回车），Go 的 HTML 解析器无法处理。

**修复**：全局替换 `\r\r\n` → `\n`。

---

## 4. 综合题 [tag_link] 缺失导致解析不折叠

**原因**：JS 查找 `[tag_link]` 作为分隔锚点，无此标记则整个解析内容始终可见。

**修复**：在每个综合题的知识标签后、解析前插入：
```markdown
[tag_link]

1）解析...
```

**关键**：`[tag_link]` 与解析之间必须空一行。

---

## 5. 表格与文本粘连

**现象**：表格后的文本被 Goldmark 吸入表格行内。

**原因**：Goldmark 将紧接表格后的文本作为表格的一部分。

**修复**：表格前后保留空行。

---

## 6. 解析内容混入下一题

**现象**：选择题 `2010-ds-011.md` 末尾出现 "组成原理 12..." 的下一题内容。

**原因**：爬虫抓取时未正确切分题目边界。

**修复**：手动删除从泄露标记到文件末尾的内容。
**检查**：`grep '组成原理 [0-9]\|操作系统 [0-9]\|计算机网络 [0-9]' content/question/*.md`

---

## 7. 留 `---` 在题目正文中

**现象**：文件末尾有 `---`，被 Hugo 解析为新的 frontmatter，导致内容截断。

**原因**：爬虫源文件的分隔符未清理。

**修复**：删除正文中多余的 `---`。

---

## 8. 2010 卷 ds-012~022 重复文件

**现象**：2010 年真题页面 Q12~Q22 每题双份渲染。

**原因**：`2010-ds-012.md` ~ `2010-ds-022.md` 的 subject 标签写为 `"组成原理"`（而非 `"计算机组成原理"`），模板按年份而非科目匹配，与 `co-*` 文件重复。

**修复**：删除这 11 个科目标签错误的文件。

---

## 9. 爬虫残留 JS 代码

**现象**：综合题解析中出现 `if(typeof window.quizDB=="undefined"){...}` 等 JS 代码。

**原因**：爬虫抓取了页面中收藏功能的 JS 代码。

**修复**：从 `if(typeof window.quizDB` 到文件末尾全部删除。

---

## 10. 常用排查命令

```bash
# 检查 [tag_link] 数量
grep -c '\[tag_link\]' content/question/2010-*.md

# 检查重复题号（文件数 vs 题号数）
ls content/question/2010-*.md | wc -l

# 检查泄露的下一题内容
grep -n '组成原理 [0-9]\|操作系统 [0-9]\|计算机网络 [0-9]' content/question/*.md

# 检查爬虫残留
grep -n '### 解答题\|## 解答题' content/question/*.md

# 检查 JS 污染
grep -l 'quizDB\|indexedDB' content/question/*.md

# 检查换行符
file content/question/2010-ds-001.md  # 显示 "CRLF" 为 Windows 格式
python -c "print(open('content/question/2010-ds-001.md','rb').read()[:100])"
```
