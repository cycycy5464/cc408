## 脚本索引

### 验证与扫描

| 脚本 | 用途 |
|------|------|
| `check_years.py` | 扫描某年份 [tag_link]/选项/空行格式问题 |
| `check_taglink.py` | 专项检查 [tag_link] 存在性 |
| `validate_content.py` | 完整性校验：选项数、答案、解析 |

| 脚本 | 用途 |
|------|------|

| `restore_missing.py` | 从爬虫 JSON 恢复缺失选项/答案 |

| 脚本 | 用途 |

## 快速开始

```bash
# 1. 扫描某年份问题
python scripts/check_years.py 2011 2012

# 2. 修复已知格式
python scripts/fix_taglink_bl.py
python scripts/fix_subq_spacing.py

# 3. 从爬虫数据恢复缺失
python scripts/restore_missing.py 2011 2012

# 4. 验证
python scripts/validate_content.py 2011 2012
```

### ❌ 已知问题

1. ~~综合题答案折叠未生效~~ — 已通过模板/JS 全局解决 ✅
2. ~~选择题选项合并~~ — 已通过 `fix_year.py` 解决 ✅
3. ~~D选项与[tag_link]之间需要空行~~ — 已通过 `fix_year.py` 解决 ✅
4. **知识链接路径失效** — 部分综合题（如 2012-co-043）内联链接使用 `../../../constitution_principle/storage/memory/content.md` 等无效路径。知识图谱结构调整后目录已失效。修复方案：只保留链接显示文字（如 `四体交叉存储模式`），删除失效的 URL 路径。
5. **题干中的代码段未用```包裹** — 如 2012-co-015 的结构体定义被误解析为多个独立 `<p>`。修复方案：用 ````c ... ```` 包裹代码段。
6. **指数上标错误** — 如 2012-co-014 选项中的 `2¹26` 实际应为 `2¹²⁶`。修复方案：使用正确的 Unicode 上标字符（⁰¹²³⁴⁵⁶⁷⁸⁹）。
7. **MD表格格式错误** — 表头与数据行挤在同一行（`|| --- | --- |`），Hugo 无法解析。修复方案：每行独立，表头分隔符单独一行。
8. **大题解析区序号混入文字行** — ① ② ③ ④ ⑤ ⑥ ⑦ ⑧ ⑨ ⑩ 等序号与后续文字在同一行，无换行。Hugo 渲染时合并成一个 `<p>` 段落，不能分行展示。修复方案：序号后加换行。
9. **大题解析区代码未包裹** — 信号量伪代码（semaphore/P/V/while/for 等）裸露在 Markdown 正文中，Hugo 当作普通段落渲染，格式混乱。修复方案：用 ```c/cpp 包裹伪代码段。
10. **解析区内容合并为一大段** — 解析的多个句子、评分说明等被写在同一行无换行，Goldmark 渲染成单一段落。修复方案：句子间/评分说明间加空行分段。
11. **选择题选项重复（新旧版本共存）** — `fix_year.py` 从爬虫覆盖选项时只追加了新选项（`A.xxx`），未删除旧的 `A\.xxx` 行，导致 7-8 个选项行。
    修复脚本：
    - `tasks/真题修复/fix_dup_opts.py`（保留 `A\.` 格式，删除 `A.` 格式）
    - `tasks/真题修复/fix_remaining_opts.py`（破损 `A\.` 文件：保留 `A.` 并转为 `A\.`）

12. **综合题 JS 代码注入** — 爬虫数据中大题解析区混入整段 JS（`window.quizDB`、`toggleSolutionDetail`、`collectAnswerQuiz`），直接写入 Markdown 文件。修复方案：截断至 `if(typeof window.quizDB=="undefined")` 标记前，去除 JS。

13. **综合题跨题污染** — 41 文件末尾含 42 题内容，42 文件末尾含 43 题内容，46 文件末尾含 47 题内容。爬虫提取时题目分隔不精确导致。修复方案：截断至对应题号标记前。

14. **LTables 数据散落** — 综合题中的表格数据（如 42 LSI 表、44 机器码表）以纯文本逐行散布，无 `|` 分隔。Hugo 渲染为独立段落而非表格。修复方案：手动构造 MD 表格格式。

15. **Frontmatter 空行** — 47 题 frontmatter 字段间有空行（`title` 后空行、`date` 后空行，etc），Hugo 解析异常。修复方案：去除 frontmatter 内空行。

16. **综合题 `[tag_link]` 重复** — 多数综合题文件有 2 个 `[tag_link]`（如 41、42、45、46、47）。修复方案：保留第 1 个，后续删除。

17. **解析区代码未被 ``` 包裹** — 解析中的伪代码/结构定义直接裸露在文本中（47 题 semaphore 伪代码、42 题 struct 定义）。Hugo 渲染为普通段落，非代码格式。修复方案：用 ```c 包裹，且代码语义可读性提升（缩进、分号、注释对齐）。

18. **大题小问解析无换行** — `1）`、`2）` 等编号与后续解析文本在同一行，无空行。Hugo 渲染为同一 `<p>`，失去分段。修复方案：编号前加 `\n\n` 空行。

19. **题目数据结构定义被仅 `|...|` 格式包裹** — 43 题解析区表格（目的网络 | 下一跳 | 接口）被当作纯文本行而非 MD 表格。修复方案：转换为标准 MD 表格（表头+分隔行+数据行）。

21. **选择题 D 选项与 [tag_link] 粘连** — D 选项行与 `[tag_link]` 之间无空行，Goldmark 合并为同一 `<p>`，D 选项内容被吸入，前端不显示。修复方案：确保 `D\.xxx` 后有一空行再接 `[tag_link]`。
22. **选择题选项行格式不统一** — 部分文件混用 `A\.`（反斜杠+点）、`A.`（纯点）、`A\`（反斜杠无点）三种格式。前端 JS 提取选项的正则只能识别前两种。修复方案：统一为 `A\.xxx` 格式。
23. **选择题 A 选项不显示（前端）** — 若 A 选项行后紧接空行再接 B 选项，或者格式异常（如 `A\xxx` 无点），前端 JS 无法提取到 A。
24. **选择题题干代码裸露未包裹** — 题干中的 C 代码段（如 `for(k=1;k<=n;k*=2)`）未被 ```c 包裹，渲染为普通段落，格式混乱。修复方案：用 ```c 包裹。
25. **解析区无内容（占位文字）** — 部分题爬虫数据中 `solutions` 和 `explanations` 为空（如 Q40、Q44），文件中只有 `解析内容...` 占位。修复方案：手动根据题干补写解析。
26. **大题表格数据散落成纯文本** — 综合题中的表格数据（LSI 表、机器码表、路由表）逐行散布，无 `|` 分隔，不形成 MD 表格。修复方案：手动构造标准 MD 表格。
27. **解析区尾部 UI 文本污染** — 解析末尾残留 `查看答案与解析  收藏` 等前端 UI 文本。修复方案：截断去除。
28. **解析区序号格式不显示** — 序号 `1）`、`2）` 等与解析文本挤在同一行无空行，Hugo 渲染为同一 `<p>`，失去分段效果。修复方案：序号前加 `\n\n` 空行。
29. **选择题 KaTeX 选项内容为空** — 爬虫数据中 KaTeX 选项只提取了 HTML 标签结构（`<span class=katex>` 等），无公式文本内容。修复方案：回退为纯文本格式。
