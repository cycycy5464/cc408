# 题目标签管理使用说明

本文档说明站点「题目标签管理」功能的使用方法。该功能位于 **标签页（`/tags/`）**，用于为真题 / 模拟题批量维护「知识点标签」，并支持按来源、科目、年份、卷等维度筛选题目，快速定位需要补标签的试题。

---

## 一、如何打开

1. 进入任意标签归档页（如点击导航的「标签」或访问 `/tags/`）。
2. 页面底部有一个折叠面板 **🛠️ 题目标签管理**，点击展开即可看到管理界面。
3. 面板打开时会自动从 `mapping/question-kp-mapping.json` 加载全部题目与现有知识点标签。

---

## 二、筛选功能

管理面板顶部有三个控件，可组合使用：

### 1. 搜索框
输入题目名称或知识点关键字，实时过滤题目列表（不区分大小写）。

### 2. 来源（`tm-filter-source`）
- **全部来源**：真题 + 模拟题
- **真题**：仅 408 真题
- **模拟题**：仅模拟题

切换来源后，「分组维度」的选项会随之变化（见下）。

### 3. 分组维度（`tm-filter-group`）
根据所选来源，提供不同的分组方式：

| 来源 | 可选分组维度 | 说明 |
| --- | --- | --- |
| 真题 / 全部来源 | 分科目、分年份、无标签题目 | 按科目或年份归类真题 |
| 模拟题 | 分科目、卷、无标签题目 | 按科目或「卷 N」归类模拟题 |

- **分科目**：按 `数据结构 / 组成原理 / 操作系统 / 计算机网络` 分组，每组标题后显示题数。
- **分年份**：按真题年份（如 `2009`、`2010`…）分组。
- **卷**：按模拟题的卷号（如 `卷1`、`卷2`…）分组。
- **无标签题目**：不分组，仅列出**尚未添加任何知识点标签**的题目，方便集中补缺。

> 提示：分组标题为「吸顶」显示，滚动长列表时始终可见当前所属分组。

---

## 三、维护知识点标签

在任一题目卡片中：

- 已有标签以胶囊形式展示，点击标签上的 **✕** 即可删除该知识点。
- 在卡片底部的输入框中输入知识点名称，点击 **添加**（或按回车）即可追加标签。
- 所有增删操作**先写入浏览器本地存储**，不会立即影响服务器数据。

---

## 四、保存与同步

面板底部操作区：

| 按钮 | 作用 |
| --- | --- |
| 💾 保存 | 将本次编辑写入浏览器 `localStorage`，仅本机生效 |
| 📥 导出映射 | 下载当前完整的 `question-kp-mapping.json`，可用于备份或离线回流到仓库 |
| 📤 导入映射 | 选择一个 `question-kp-mapping.json` 文件覆盖本地数据 |
| ☁ 同步到仓库 | **仅当配置了远端代理端点（`params.tagProxyEndpoint`）后出现**，将编辑提交回 GitHub 仓库并触发自动重新部署 |

### 本地存储说明
- 标签编辑默认保存在当前浏览器的 `localStorage`（键名 `cc408-kp-edits`），换设备 / 清缓存会丢失。
- 若需让修改对**所有人永久生效**，请使用「☁ 同步到仓库」（需管理员部署 Serverless 代理，详见 `06-serverless-proxy-implementation.md`）；否则请「📥 导出映射」后提交到仓库。

---

## 五、常见问题

**Q：为什么「☁ 同步到仓库」按钮不显示？**
A：该功能依赖 `config/_default/hugo.yaml` 中的 `params.tagProxyEndpoint`。未配置时仅支持本地保存与导入/导出。

**Q：分组维度里没有「分年份」或「卷」？**
A：选项会随「来源」自动切换——真题显示「分年份」，模拟题显示「卷」。确认来源选择是否正确。

**Q：改了标签但页面没更新？**
A：删除 / 添加是即时刷新卡片的；但「保存」只是本地落盘。若要在站点上生效，需同步到仓库（或导出后由管理员合入）。
