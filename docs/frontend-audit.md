### 维度一：设计系统（Design System）

#### D1-P0｜真题页「浅色孤岛」—— 变量未接入设计系统
- **严重度**：P0 → ✅ 已修复
- **影响**：在整体深色站点中，真题内容区出现成片浅色块，风格割裂
- **状态**：已在本会话修复

#### D1-P1｜科目配色双来源——JS 与 SCSS token 漂移
- **严重度**：P1
- **影响**：同一科目在不同位置显示不同色
- **证据**：
  - `knowledge-graph.js:10-16` 硬编码 `data-structure:#58d6c0 / computer-org:#f59e0b / os:#8b5cf6 / network:#3b82f6`
  - `custom.scss:13-16` 定义 `--color-ds:#39bae6 / --color-co:#ff8f40 / --color-os:#a277ff / --color-cn:#95e6cb`
  - 四科**全部不一致**，且键名体系不同

### 维度二：UX 流程（UX Flow）

#### D2-P0｜资源筛选器完全失效——选择器与模板不匹配
- **严重度**：P0（核心功能不可用）
- **影响**：资源下载页的筛选无任何效果
- **证据**：
  - `resource-filter.js` 依赖 `#filter-search`、`#filter-subject`、`#filter-difficulty` 与 `.card-glass[data-subject]`
  - `layouts/resources/list.html` 只渲染 `<div class="card">`，无 `card-glass`、无 `data-subject`/`data-difficulty`
  - 全仓搜索 `card-glass` = **0 匹配**

#### D2-P1｜两套 Toast 实现风格不统一
- **严重度**：P1
- **影响**：收藏提示与题集提示的样式来源、配色、动画各不相同；`collect-quiz.js` 缺 error 态红色
- **证据**：
  - `collect-quiz.js` 用 `el.style.cssText` **内联**硬编码样式
  - `quiz-collection.js` 用 CSS 类 `.qc-notification`，含 `@keyframes` 滑入与 `.error{background:#e74c3c}`

### 维度三：UI 概念与布局（UI Concept / Layout）

#### D3-P1｜缺少明暗主题选项 → ✅ 已修复
#### D3-P2｜图谱信息面板固定定位可能遮挡内容
- **严重度**：P2
- **影响**：桌面中等宽度（769–1100px）易遮挡
- **改进**：面板支持拖拽/折叠，或关闭后「不自动再弹」的记忆开关

### 维度四：实现与无障碍（Implementation & Accessibility）

#### D4-P1｜自定义控件缺 ARIA 角色与键盘可达性
- **严重度**：P1
- **影响**：刷题选项、图谱分页 Tab 纯靠鼠标
- **证据**：
  - `exam-option`：无 `role`/`aria-checked`/`tabindex`/键盘事件
  - `graph-tab`：无 `role="tab"`/`aria-selected`/键盘
  - `reveal-btn`：缺 `aria-expanded`
  - `qc-mode-btn`/`filter`/`view-btn`：缺 `aria-pressed`

#### D4-P1｜动效无 `prefers-reduced-motion` 守卫
- **严重度**：P1
- **影响**：卡片 hover 位移、滑入动画等对所有用户强制播放
- **证据**：全仓搜索 `prefers-reduced-motion` = 0 匹配

#### D4-P2｜JS 无障碍属性系统性缺失
- **严重度**：P2
- **影响**：除 `kpAddInput` 回车添加外，无任何 `role/aria-label/tabindex/键盘事件` 管理
