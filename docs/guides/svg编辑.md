## SVG 文字位置调整工具 — 使用文档

### 概述

这是一个用于调整 SVG 题目配图中**文字标签位置**的可视化工具，位于 `static/svg-editor.html`。解决从 draw.io 导出的 SVG 里 `foreignObject` → `<text>` 转换后文字位置不精确的问题。

### 访问方式

```
http://localhost:1313/cc408/svg-editor.html
```

前提：Hugo dev server 正在运行（`hugo server`）。

### 核心功能

| 功能         | 操作                        | 说明                                                     |
| ------------ | --------------------------- | -------------------------------------------------------- |
| **选择题目** | 下拉列表选取 SVG 文件       | 覆盖 2009-2025 年真题配图，按 `年份_Q题号_序号.svg` 命名 |
| **拖拽调整** | 在预览区拖拽文字标签        | 实时调整 `x`/`y` 坐标                                    |
| **表格编辑** | 右侧坐标表中直接输入数值    | 适合精确微调                                             |
| **全局偏移** | 设置 X/Y 偏移量后点击"应用" | 对所有文字统一移动                                       |
| **保存**     | 点击 `💾 保存` 按钮          | 两种回写方式（见下方）                                   |
| **重置**     | 点击 `↺ 重置`               | 重新从服务器加载，丢弃所有修改                           |

### 保存方式

#### 方式一：自动保存（推荐）
需启动保存服务：
```bash
node scripts/save-server.js
```
点击保存后，文件直接写入 `static/images/questions/{文件名}`。

#### 方式二：手动下载
若 `save-server.js` 未运行，浏览器会**自动下载修改后的 SVG 文件**，需手动复制到 `static/images/questions/` 覆盖原文件。

### 工作原理

1. 加载 SVG → 解析 `<text>` 元素 → 提取文字坐标
2. 在 SVG 上方覆盖 `<div>` 文字层（绝对定位），映射 viewBox 坐标到屏幕像素
3. 拖拽时实时更新坐标，保存时用正则替换回 SVG 源文件中的 `x`/`y` 属性
4. 保存也会自动补全 `font-size="16" fill="#000"` 并移除 `opacity` 属性

### 注意事项

- **只支持含 `<text>` 元素的 SVG**。如果 SVG 使用 `foreignObject` 渲染文字（draw.io 导出格式），需要先用 `scripts/batch-fix-svgs.js` 转换为 `<text>`
- 保存到 `static/` 后，下次 Hugo 构建才会同步到 `public/`
- 修改过的 SVG 文件是用户手动内容，**遵循 Lesson #15**——脚本不应覆盖用户已手动调好的文件

### 相关脚本

| 脚本                        | 用途                                                        |
| --------------------------- | ----------------------------------------------------------- |
| `scripts/batch-fix-svgs.js` | 批量将 `foreignObject` + fallback `<image>` 转换为 `<text>` |
| `scripts/save-server.js`    | 接收编辑器 POST 请求，写入 `static/images/questions/`       |
