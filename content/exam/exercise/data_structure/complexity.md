##### 1

函数 `fun()` 的时间复杂度是多少？（ ）

```c
int fun(int n) {
    int count = 0;
    for (int i = n; i > 0; i /= 2)
        for (int j = 0; j < i; j++)
            count += 1;
    return count;
}
```

正确答案：C
这是一个嵌套循环，外层是一个 以 2 为因子的递减（即对数级别），内层是线性地跑 `j < i`。
我们可以将整个过程看成：
- 第1轮：`i = n`，执行 `n` 次
- 第2轮：`i = n/2`，执行 `n/2` 次
- 第3轮：`i = n/4`，执行 `n/4` 次
- &mldr;
- 直到 `i <= 0`
因此，总执行次数是：
n+2n​+4n​+⋯+1
这是一个等比数列，其和趋近于：
n(1+21​+41​+⋯)=2n
所以函数 `fun(n)` 的时间复杂度是 O(n)。

##### 2

函数 `fun()` 的时间复杂度是多少？

```c
int fun(int n) {
    int count = 0;
    for (int i = 0; i < n; i++)
        for (int j = i; j > 0; j--)
            count = count + 1;
    return count;
}
```

正确答案：B
通过计算表达式 `count = count + 1;` 的执行次数可以得出时间复杂度。该表达式的执行次数为 0 + 1 + 2 + 3 + 4 + &mldr; + (n-1) 次。因此时间复杂度为：
O(0+1+2+3+⋯+n−1)=O(2n(n−1)​)=O(n2)

##### 3

汉诺塔问题中，n 个圆盘的最优时间递推关系式是（ ）

正确答案：D
解决汉诺塔问题的递归步骤如下。设三根柱子分别为 A、B 和 C，目标是将 n 个圆盘从 A 移动到 C。
具体步骤为：
1. 将 n-1 个圆盘从 A 移动到 B，此时 A 上仅剩第 n 个圆盘
2. 将第 n 个圆盘从 A 移动到 C
3. 将 n-1 个圆盘从 B 移动到 C，叠放在第 n 个圆盘之上
上述递归解法的时间复杂度递推函数 T(n) 可表示为：
T(n)=2T(n−1)+1

##### 4

下面函数的时间复杂度是多少？

```c
void fun(int n, int arr[]) {
    int i = 0, j = 0;
    for (; i < n; ++i)
        while (j < n && arr[i] < arr[j])
            j++;
}
```

正确答案：A
- 关键观察：变量 `j` 在整个函数执行过程中仅初始化一次，不会随 `i` 的变化而重置。
- 行为分析：
内层 `while` 循环的总迭代次数取决于 `j` 的移动方向。由于 `j` 始终递增且不超过 `n`，其最大累计移动次数为 `n` 次。
- 与变体对比：
若将函数改为以下形式（`j` 被重置为 0）：`void fun(int n, int arr[]) { int i = 0, j = 0; for (; i < n; ++i) { j = 0; while (j < n && arr[i] < arr[j]) j++; } }`此时内层循环会因每次 `i` 变化而重新遍历数组，导致时间复杂度退化为 O(n²)。
- 结论：原函数中 `j` 的单向移动特性使得整体时间复杂度为 O(n)。

##### 6

当说算法 X 比 Y 渐近更高效时，其含义是什么？（ ）

正确答案：B
当我们说算法 X 比 Y 渐近更高效时，意味着随着算法输入规模的不断增大，X 的运行时间最终会比 Y 的运行时间更快。这通常通过大 O 表示法（big O notation）来描述，它给出了算法最坏情况运行时间的上界。具体来说：
- 如果 X 比 Y 渐近更高效，则存在一个常数 c，使得对于所有足够大的输入，X 的运行时间 ≤ c × Y 的运行时间；
- 即 X 的大 O 复杂度类低于 Y（如
O(n)
vs
O(n2)
）；
- 这表明在 大输入情况下，X 在效率上始终优于 Y；
- 然而，对于 小输入，Y 可能仍然比 X 更快（因常数因子或低阶项影响）。
因此，我们不能断言 X 是所有输入情况下的更好选择，但可以说 X 是除可能的小输入外所有输入情况下的更好选择。

```c
int unknown(int n) {
    int i, j, k = 0;
    for (i  = n/2; i <= n; i++)
        for (j = 2; j <= n; j = j * 2)
            k = k + n/2;
```

##### 10

当使用二分查找计算待插入数据的位置时，插入排序的最坏情况时间复杂度是多少？（ ）

正确答案：C
使用二分查找确定插入位置不会降低插入排序的时间复杂度。这是因为插入操作包含两个步骤：
1. 计算插入位置
  - 使用二分查找可将此步骤时间复杂度从 O(N) 降低至 O(log N)
2. 将插入位置后的数据向右移动一位
  - 此步骤始终需要 O(N) 时间复杂度
尽管第一步通过二分查找优化了时间效率，但第二步的数据移动操作仍是决定性因素。由于每次插入都需要进行一次完整的数组移动（最坏情况下），总共有 N 次插入操作，因此整体时间复杂度仍为 O(N²)。

##### 14

正确答案：D

logn

##### 29

当以下递归函数被调用时，n=5 的输出是什么？（ ）

```cpp
int foo(int n) {
    if (n == 0)
        return 1;
    else
        return n * foo(n - 1);
}
```

正确答案：C
给定的递归函数计算非负整数 n 的阶乘。当函数以 n=5 调用时，函数会检查 n 是否等于 0。由于 n 不等于零，它返回 n 乘以将 n-1 作为输入调用相同函数的值。递归调用会持续进行直到达到基准情况（n==0）。foo(5) 的计算过程如下：
- `foo(5) = 5 * foo(4)`
- `foo(4) = 4 * foo(3)`
- `foo(3) = 3 * foo(2)`
- `foo(2) = 2 * foo(1)`
- `foo(1) = 1 * foo(0)`
- `foo(0) = 1`
逐步代入计算：
1. 将 `foo(0)=1` 代入 `foo(1)`:
`foo(1) = 1 * 1 = 1`
2. 将 `foo(1)=1` 代入 `foo(2)`:
`foo(2) = 2 * 1 = 2`
3. 将 `foo(2)=2` 代入 `foo(3)`:
`foo(3) = 3 * 2 = 6`
4. 将 `foo(3)=6` 代入 `foo(4)`:
`foo(4) = 4 * 6 = 24`
5. 将 `foo(4)=24` 代入 `foo(5)`:
`foo(5) = 5 * 24 = 120`
因此，当以 n=5 调用该函数时，输出为 120，即选项 C。
